## вњЁ Features

- **Conversational Data Analysis:** Chat directly with your data using Gemini.
- **Automated Data Cleaning:** The system handles messy data, standardising columns and types behind the scenes.

## рџљЂ Getting Started

### 1. Prerequisites
- **Python 3.10+**
- A **Google Gemini API Key** вЂ” get one from [Google AI Studio](https://aistudio.google.com/).

### 2. Setup your Environment

Create a `.env` file in the root directory:
```env
GEMINI_API_KEY=\"your-api-key-here\"

# Supabase configuration
SUPABASE_URL=\"your-supabase-url\"
SUPABASE_ANON_KEY=\"your-anon-key\"
SUPABASE_SERVICE_ROLE_KEY=\"your-service-role-key\"

# Optional settings
FLASK_DEBUG=False
PORT=5000
MAX_UPLOAD_MB=1
```

### 3. Running the App (Windows)

Just double-click the `start.bat` file!

This will automatically:
1. Check if Python is installed.
2. Install the necessary dependencies from `requirements.txt`.
3. Start the Flask backend server.
4. Launch the web application in your default browser.

**Manual Start:**
```bash
# 1. Create a virtual environment (recommended)
python -m venv .venv
.venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Start the server
python server.py
```
Then navigate to `http://localhost:5000` in your web browser.

## рџ”’ Security

The code executor (`code_executor.py`) runs AI-generated Python on your machine with the following safeguards:

- **AST validation** вЂ” code is parsed and checked before execution
- **Module/attribute allowlists** вЂ” only safe builtins (`pd`, `np`, `len`, `range`, etc.) are permitted
- **Blocked modules** вЂ” `os`, `sys`, `subprocess`, `socket`, `pathlib`, `shutil`, `ctypes`, etc.
- **Subprocess isolation** вЂ” code runs in a spawned child process, not the main server
- **60-second timeout** вЂ” prevents runaway execution
- **10,000 character limit** вЂ” prevents dataset hardcoding in generated code
- **Cloud Storage** вЂ” datasets are stored securely in Supabase Storage instead of the local filesystem
- **API Authentication** вЂ” routes are protected by Supabase JWT verification (`@require_auth`)
- **XSS prevention** вЂ” all rendered content sanitised with DOMPurify

> **Warning:** This application is designed as a local development tool. Never expose it to the public internet without adding authentication, a production WSGI server (e.g., Waitress or Gunicorn), and stronger sandboxing (e.g., Docker containers).
